# AGENT.md — Contrato del Sistema

Este vault es la mente del agente. No es documentación.

## Reglas operativas

1. Leer ACTIVE_CONTEXT.md al inicio de toda sesión — sin excepciones.
2. Navegar wiki SIEMPRE por index.md primero — nunca leer carpetas ciegamente.
3. Actualizar ACTIVE_CONTEXT.md al finalizar trabajo significativo.
4. Todo error documentado en ERRORS_LOG.md antes de seguir.
5. Respuestas concisas — el detalle vive en el vault, no en el chat.
6. Wikilinks con doble corchete para todos los enlaces internos.
7. Registrar en log.md toda operación estructural.

## Estructura del vault

- ACTIVE_CONTEXT.md — RAM del proyecto
- PROJECT_MAP.md — mapa del ecosistema
- INBOX.md — captura bruta (no leer automáticamente)
- ERRORS_LOG.md — aprendizaje de errores
- Tasks/index.md — estado de tareas
- wiki/ — conocimiento organizado por índice

## Tecnologías del proyecto

- React 19, CSS Modules, jsPDF, html2canvas
- Vercel (deploy automático desde main)
- GitHub (branches: main, preprod, lab)
